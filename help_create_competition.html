Create a Competition

CodaLab competitions provide an opportunity for researchers, developers and algorithmists to create solutions for problems across a wide range of domains, and advance the state of the art for their respective areas of interest. This guide covers everything you need to know to create and manage a CodaLab competition.

Note: CodaLab currently supports the following browsers:

  • Internet Explorer
  • Google Chrome
  • Mozilla Firefox

Competition Overview

Competitions are conducted in two phases: training and competition. The appropriate data is made available to participants at each phase of the competition. During the training phase, participants have access to training data to refine their algorithms. During the competition phase, participants are provided with competition data to generate results which they can then submit to the competition.

To create a competition, you need to create and upload a Competition Bundle. A Competition Bundle is a .zip archive which contains a YAML file that describes the competition and its assets. You can download example competition bundles here: https://github.com/codalab/competition-examples. For this tutorial, we will be using the example_pi example competition.

The files that make up a competition bundle are as follows:

Item Description
competition.yaml Lists all of the assets for a competition.
data.html Contains links to the competition data.
evaluation.html Explains the metrics and scoring methods for the challenge.
overview.html Describes the goals of the competition.
terms_and_conditions.html Lists the terms for participating in the challenge (for example data licensing and results submission).
program.zip Contains the evaluation program which compares the submitted results with the answers and passes the scores to the CodaLab leaderboards.
reference.zip Contains reference data used to score the competition.
submission.zip Contains an example of how to format submissions. This is for demonstration purposes only and does not need to be included in the submission.
logo.jpg A logo or image for the competition.

Create a Competition Bundle

In this section you will learn how to create a competition bundle using an example competition bundle for guidance. The goal of the example competition is to compute the value of pi (3.14...). Each participant submits a single float value and the submission that contains the closest value of pi is the winner.

Bundle File Structure

Competition bundles are structured as follows:

competition.zip
  |- competition.yaml
  |- data.html
  |- evaluation.html
  |- logo.jpg
  |- overview.html
  |- program.zip
  |- reference.zip
  |- submission.zip
  |- terms_and_conditions.html

Note: There are no .zip archives in the example competition as downloaded. This is so that the files can be properly downloaded. You will need to manually create .zip archives from the program, and reference folders.

Create a YAML File

The manifest file (competition.yaml) defines all aspects of the competition, and is defined using YAML.

IMPORTANT: When setting timestamps, you must enter time values in Coordinated Universal Time (UTC). The following examples show the preferred date/time formats to use:

  • YYYY-MM-DD
    For example: end_date: 2014-03-01 specifies a competition end date of March 1st, 2014 at midnight UTC.
  • YYYY-MM-DD HH-MM
    For example: start_date: 2014-03-01 18:15 specifies a start date of March 1st, 2014 at 6:15 PM UTC.

Here's the example competition.yaml:

# Build an example competition
---
title: Example Competition
description: This is an example competition to demonstrate the competition bundle system. In this competition, the objective is to see who can get closest to Pi.
image: logo.jpg
has_registration: True
end_date: 
html:
    overview: overview.html
    evaluation: evaluation.html
    terms: terms_and_conditions.html
    data: data.html
phases:
    1:
        phasenumber: 1
        label: "Test"
        start_date: 2014-05-01
        max_submissions: 100
        scoring_program: program.zip
        reference_data: reference.zip
        datasets: 
            1:
                name: Test Data
                url: http://www.bing.com/
                description: Testing data set.
            2:
                name: Final Data
                url: http://www.bing.com/
                description: Competition data set.
    2:
        phasenumber: 2 
        label: "Training"
        start_date: 2014-06-01
        max_submissions: 3
        scoring_program: program.zip
        reference_data: reference.zip
    3:
        phasenumber: 3
        label: "Challenge"
        start_date: 2014-07-01
        max_submissions: 3
        scoring_program: program.zip
        reference_data: reference.zip
leaderboard:
    leaderboards:
        Results: &RESULTS
            label: Results
            rank: 1
    columns:
        Difference:
            leaderboard: *RESULTS
            label: Difference
            numeric_format: 6

Add Folder Contents to ZIP Archives

To make this example complete it's important to understand how to build and package the program.zip and reference.zip referred to in the competition.yaml.

The program.zip bundle contains the program that compares the users submission with the reference data (in the reference.zip bundle) to score the submission. In this case the reference data contains the value of pi. The program.zip bundle computes the absolute difference of the submitted value from the reference value.

Here are the contents of the reference.zip file: 

reference.zip 
  |- answer.txt (Contains: 3.14159265359)
  |- metadata   (Contains: This is the authoritative result.)

Here are the contents of the program.zip file

program.zip
  |- evaluate.py (The evaluation program code)
  |- setup.py (Enables py2exe to build a windows executable from evaluate.py)
  |- metadata (Contains a definition of the command syntax, and descriptive text)
  |- readme.txt (Contains information explaining the evaluation program)

1. Add the contents of the program, reference, and submission folders to individual ZIP archives. Using our example competition files, zip the contents of the following folders: 

C:\example_pi\program -> program.zip
C:\example_pi\reference -> reference.zip

2. Create a .zip archive of C:\MyCompetition. This will be known as the "competition bundle".

Upload a Competition Bundle

Follow these steps to upload the competition bundle to CodaLab.

  1. Log in to your CodaLab account.
  2. Navigate to the My CodaLab page.
  3. Click the Competitions I'm Running tab.
  4. Click Create Competition.
  5. Click Upload definition..., locate the competition bundle you created, select it and click Open. The file will be uploaded to CodaLab.
  6. You can manage the competition from your CodaLab Dashboard (the My CodaLab page).
